I read a tutorial on how to set aliases for different commands including git commands...

However, I found this post by Kurt Dowswell to be much more effective since it keeps the changes forever and I don't have to keep typing 'source ~/.custom_aliases' every time I open bash. I also like how extensive his list is and that I could add one for 'git push origin master' wherever I liked. I also liked how easy he made it to list out all the aliases in Git Bash if I ever forgot the ones that were set. So I have officially followed his step by step and am very pleased with my new git aliases! My personal favorite is 'g ac' so I can type my commit message after that without having any errors like with the .custom_aliases function with a placeholder. Yay for progress in improving my environment for better productivity!

Here is a screenshot of all the aliases from Kurt Dowswell with some of my own mixed in. I'm not sure why some of them are repeated, but that is research for another day!



Here is the starter code for anyone reading this to try it out. But I encourage lots of research before officilly changing any config files or keep a copy somewhere in case you want to go back to the original config file! 

    [filter "lfs"]
        clean = git-lfs clean -- %f
        smudge = git-lfs smudge -- %f
        process = git-lfs filter-process
        required = true
    [user]
        name = enter your name here
        email = enter your email here

    [alias]
    # logs
    l = log --pretty=format:"%C(yellow)%h\\ %ad%Cred%d\\ %Creset%s%Cblue\\ [%cn]" --decorate --date=short
    lg='git log --graph --oneline --decorate --all'

    ap = add -p
    aa = add .
    ac = !git add -A && git commit -m

    ca = commit -a --verbose
    cm = commit -m
    cam = commit -a -m
    m = commit --amend --verbose
